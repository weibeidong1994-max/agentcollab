# AgentHub — 技术方案（v3）

> 更新日期：2026-04-20
> 基于 v2 方案 + 第一天实际开发实现的技术细节

---

## 1. 技术栈（最终确定）

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 桌面壳 | **Tauri 2** | 2.x | 跨平台桌面应用框架 |
| 前端 | **React + TypeScript** | React 19, TS 5.x | UI 渲染 |
| 终端渲染 | **xterm.js** | @xterm/xterm + addons | 嵌入式终端模拟器 |
| PTY 后端 | **Rust (portable-pty)** | 0.8 | 伪终端，支持交互式 CLI |
| 数据库 | **SQLite (tauri-plugin-sql)** | tauri-plugin-sql | 元数据持久化 |
| 文件 I/O | **Rust 自定义命令** | std::fs | 写入 context.md 等 |
| 包管理 | npm (前端) + cargo (后端) | - | 分层管理依赖 |
| 构建工具 | Vite (前端) + cargo (Rust) | - | 开发热更新 |

### 与 v2 技术栈的差异

| v2 设想 | v3 实际 | 原因 |
|---------|---------|------|
| 纯 TypeScript | **Rust + TypeScript** | PTY 需要系统调用，必须用 Rust |
| Node.js 后端 | **无独立后端** | Tauri 的 Rust 后端足够 |
| child_process.spawn | **portable-pty** | 需要 PTY 支持 TUI 应用 |
| better-sqlite3 | **tauri-plugin-sql** | WebView 不能用 Node.js 模块 |
| sqlite-vec + ONNX | **暂未引入** | P0 阶段不需要向量检索 |

### 2.2 数据流

```
用户键盘输入
  → xterm.onData()
  → ptyWrite(sessionId, data) [Tauri IPC]
  → Rust: write to PTY master
  → Agent CLI 收到输入

Agent CLI 输出
  → PTY slave → PTY master
  → Rust: read from PTY, base64 encode
  → emit "pty-output" event [Tauri Event]
  → Frontend: listenPtyOutput() callback
  → xterm.write(data)
  → 同时累积到 outputBufferRef

定时蒸馏 (每 10s setInterval)
  → 遍历 terminalRefs
  → handle.getOutput() 获取完整输出
  → extractFromTerminal(output) 模式匹配提取
  → 去重 → 写入 SQLite (memories 表)
  → buildContextMd() 生成 Markdown
  → writeFile(context.md) [Tauri IPC]
  → Rust: 写入 {project}/.agenthub/context.md

上下文注入
  → 用户点击「注入上下文」按钮
  → handle.writeToPty("请阅读项目上下文文件: {path}\n")
  → Agent CLI 收到指令
  → Agent 读取 context.md 文件
```

### 3.2 关键文件职责

#### `src-tauri/src/lib.rs` — Rust 后端核心

**PTY 命令**：
- `pty_spawn`：创建 PTY fork 子进程执行 Agent CLI
- `ptyWrite`：向 PTY 写入数据（用户输入）
- `ptyResize`：调整终端大小
- `ptyKill`：终止进程

**文件命令**：
- `writeFile`：写入文件（用于 context.md）
- `readFile`：读取文件
- `mkdirp`：递归创建目录

**关键实现细节**：
```rust
// PTY 输出通过 Tauri Event 发送到前端（base64 编码）
app.emit("pty-output", Payload {
    id: session_id.clone(),
    data: base64_engine.encode(&buf),  // Uint8Array -> base64 string
})?;

// 只有 wait 线程发送 exit 事件（避免 reader 先发 unknown）
let status = child.wait()?;
app.emit("pty-exit", Payload {
    id: session_id.clone(),
    exit_code: Some(status.code()),
})?;
```

#### `src/components/TerminalPanel.tsx` — 终端组件

**暴露接口（useImperativeHandle）**：
```typescript
export interface TerminalPanelHandle {
    writeToPty: (text: string) => void;   // 注入文本到 PTY
    getOutput: () => string;              // 获取完整终端输出（已清洗 ANSI）
}
```

**生命周期**：
1. `useEffect([agentId])`：创建 Terminal + FitAddon + WebLinksAddon
2. 异步初始化：获取 homeDir → 配置环境变量(含 PATH) → `ptySpawn()` 启动 Agent
3. 监听 `ptyOutput` 事件 → `xterm.write()` + 累积到 `outputBufferRef`
4. 监听 `ptyExit` 事件 → 显示退出码
5. `xterm.onData` → `ptyWrite()` （用户键盘输入转发给 PTY）
6. 清理函数：`resizeObserver.disconnect()` + `unlistenAll` + `ptyKill()` + `xterm.dispose()`

**Agent 配置**：
```typescript
const agentConfigs = {
    "claude-code": { command: "/Users/bytedance/.npm-global/bin/claude", buildArgs: () => [] },
    kimicode:      { command: "/Users/bytedance/.local/bin/kimi", buildArgs: () => [] },

```

#### `src/App.tsx` — 主应用 + 蒸馏引擎

**状态管理**（全部 useState）：
- `projects`, `activeProjectId` — 项目列表和当前项目
- `conversations`, `activeConvId` — 对话列表和当前对话
- `memories` — 当前项目的记忆列表
- `agents` — 已安装的 Agent 列表
- `activeAgentId` — 当前对话绑定的 Agent
- `distillStatus`, `contextFilePath` — UI 状态

**蒸馏引擎 `extractFromTerminal()`**：

从原始终端输出中按行扫描，多维度模式匹配：

```typescript
interface ExtractedInfo {
    userMessages: string[];     // 用户需求（› 提示符 / 中文意图词）
    fileOperations: string[];   // 文件操作（Wrote/Created/Modified + 文件路径匹配）
    agentActions: string[];     // Agent 执行内容（●/•/数字列表项）
    codeBlocks: string[];       // 代码块预览
    keyDecisions: string[];     // 技术决策（决定/选择/采用关键词）
}
```

匹配规则优先级：
1. `Wrote/Created/Modified X lines to file` → fileOperations
2. `› text` → userMessages（Claude Code 提示符）
3. 中文意图词开头（帮我/请/需要/想要...）→ userMessages
4. `●/•/→/-` 或 `N.` 开头 → agentActions
5. 决策关键词 → keyDecisions
6. 文件扩展名匹配 (.ts/.html/.py...) → fileOperations
7. ``` 代码块 → codeBlocks

**context.md 生成 `buildContextMd()`**：

**自动蒸馏调度 `autoDistill()`**：
- 每 10 秒执行一次（`setInterval`）
- 遍历所有 `terminalRefs` 中的活跃终端
- 检查输出是否有新增内容（`lastOutputLenRef` 比较）
- 有新内容时：提取 → 去重写入 DB → 更新 UI → 写 context.md
- 无新内容时：只同步 context.md 时间戳

## 4. Tauri 权限配置（关键！）

### 4.1 capabilities/default.json

这是最容易出错的地方。Shell scope 必须包含所有 Agent 的绝对路径：

```json
{
  "$schema": "../gen/schemas/desktop-schema.json",
  "identifier": "default",
  "description": "Default capability for the main window",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "shell:allow-spawn",
    "shell:allow-execute",
    "shell:allow-kill",
    "opener:default"
  ],
  "permissions": [
    {
      "identifier": "shell:allow-spawn",
      "scopes": [
        { "name": "shell", "cmd": "/bin/bash", "args": true },
        { "name": "claude", "cmd": "/Users/bytedance/.npm-global/bin/claude", "args": true },
        { "name": "kimi", "cmd": "/Users/bytedance/.local/bin/kimi", "args": true },
        { "name": "hermes", "cmd": "/Users/bytedance/.hermes/hermes-agent/venv/bin/hermes", "args": true },
        { "name": "openclaw", "cmd": "/Users/bytedance/.npm-global/bin/openclaw", "args": true }
      ]
    },
    {
      "identifier": "shell:allow-execute",
```

## 5. 已解决的问题记录

### 5.1 数据库相关

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `Cannot read properties of undefined (reading 'invoke')` | tauri-plugin-sql 未正确加载 | 改用 tauri-plugin-sql 替代 better-sqlite3 |
| `UNIQUE constraint failed: projects.id` | React StrictMode 双挂载导致重复插入 | `_initPromise` 单例 + `INSERT OR IGNORE` |
| `UNIQUE constraint failed: tasks.id` | 任务 ID `t0/t1` 硬编码冲突 | 改用 `crypto.randomUUID()` |
| `no such column: category` | 表结构变更后旧 schema 残留 | 初始化时先 `DROP TABLE IF EXISTS` |
| `FOREIGN KEY constraint failed` | 删除表顺序错误（memories 引用 conversations） | 按依赖顺序删除：memories → conversations → projects |

### 5.2 Shell/CLI 相关

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `shell.execute not allowed` | 缺少 `shell:allow-execute` 权限 | 在 capabilities 中添加 |
| `Scoped command not found` | Tauri Shell 需要显式 scope（用 name 不是 cmd） | 添加 scope 条目，使用 name 字段 |
| Agent 全部 offline | PATH 缺少 `~/.local/bin` 和 `~/.npm-global/bin` | 启动时注入完整 PATH |
| `No such file or directory (os error 2)` | CLI 名称错误（`claudecode` vs `claude`） | 使用正确的 CLI 命令名 |
| Hermes 路径错误 | 需要 venv 路径 | `.hermes/hermes-agent/venv/bin/hermes` |

### 5.3 PTY/终端相关

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 终端卡住无反应 | Tauri Shell `Command.spawn()` 不提供 PTY | 引入 portable-pty Rust crate |
| Exit code "unknown" | reader 线程在 wait 线程前发送了 exit 事件 | 只让 wait 线程发送 pty-exit |
| `child.wait()` borrow 错误 | `child` 非 mut | 声明为 `let mut child` |
| `and_then` 类型不匹配 | `s.exit_code()` 返回 u32 非 Option | 用 `.map(\|s\| s.exit_code())` |
| 用户看不到执行过程 | v2 设计是后台执行 | 改为嵌入式 xterm.js 终端 |

### 5.4 蒸馏相关

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| context.md 为空 | 正则匹配无法覆盖实际终端输出 | 改为多维度模式匹配 |
| 记忆完全没有同步 | `getConversationTurns()` 返回空数组 | 轮次追踪对 TUI Agent 不可靠，改用原始输出 |
| 提取的内容看不懂 | 只有过滤后的碎片行 | 结构化分类：用户需求/文件操作/执行内容/技术决策 |
| 不知道保存在哪里 | 内存数据库不可见 | 双写：SQLite + 文件系统 `{project}/.agenthub/context.md` |

### 5.5 UX 相关

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 点击任务后无反应 | `currentAgentRef` 阻止重新执行 | 用 `key={activeConvId}` 强制 remount |
| 切换对话后 Agent 丢失 | `handleSelectConversation` 未恢复 agent_id | 添加 agent_id 恢复逻辑 |
| prompt() 阻塞 | Tauri WebView 不支持 | 替换为内联表单输入 |
| 终切出去再切回来记录消失 | `key={}` 导致 remount 销毁终端 | 待修复：改用 CSS display:none |
