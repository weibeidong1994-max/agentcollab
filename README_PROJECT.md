# AgentHub — 多 Agent 协作工作台

**GitHub**: https://github.com/weibeidong1994-max/agentcollab

## 解决了什么问题

**问题**：当你同时使用多个 AI 编程助手（如 KimiCode、Claude Code）时，它们之间互不知情，无法共享上下文。导致：

- 同一项目的需求在多个 Agent 间重复解释
- 不同 Agent 的决策互相矛盾
- 项目记忆无法积累，每次对话都要重新建立上下文

## 解决方案

AgentHub 是一个**本地优先的多 AI Agent 上下文管家**，通过以下方式解决：

1. **嵌入式终端** — 在同一窗口内启动多个 Agent 的交互式终端，支持 Claude Code、KimiCode、Hermes 等
2. **自动记忆蒸馏** — 直接读取 Agent 本地日志（`~/.kimi/sessions/`、`~/.claude/projects/`），精准提取用户需求、思考过程、技术决策
3. **上下文注入** — 将项目记忆以 Markdown 文件形式注入到 Agent 的对话中
4. **跨会话聚合** — 所有对话的记忆按类别全局聚合，避免信息碎片化

## 运行流程

### 1. 项目管理
```
创建项目 → 选择 Agent → 启动终端会话
```

### 2. 记忆蒸馏
```
Agent 运行时 → 定时读取本地日志 → LLM 提取五维信息
```

### 3. 上下文注入
```
新对话开始 → 选择项目记忆 → 生成 context.md → 注入到 Agent 终端
```

```

```

## LLM 蒸馏配置

点击左上角 ⚙️ 按钮配置：
- **API Key**：智谱开放平台 API Key
- **API URL**：`https://open.bigmodel.cn/api/paas/v4/chat/completions`
- **模型**：`glm-4-flash`（默认）或 `glm-4.7-flash`
